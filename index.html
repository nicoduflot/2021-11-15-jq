Problèmes CSS courants

Le calcul par défaut des tailles en CSS est content-box, il est conseillé de le passer en border-box

Les tailles calculées : si vous voulez utiliser des cadratin (em), il est plus sûr d'utiliser les cadratins racine (rem) car il sont toujours calculés par rapport au cadratin racine du user stylesheet et non pas le cadratin calculé du conteneur.

Les polices de caractère

On utilisera les propriétés @font-face pour déclarer en CSS une police de caractère dont les fichiers (eot, svg, ttf, woff, woff2) sont hébergés sur le site pour ne pas forcer l'utilsateur à télécharger et installer des fichiers qui sont peut-être corrompus ou porteur de virus.

HTML - CSS sur le texte

Il existe quelques balises pour changer directement l'affichage du texte en HTML. Un texte en gras (b), n'a aucune influence sur le SEO. si on utilise strong, le contenu de la balise influe sur le SEO.

C'est pareil avec l'italique Italique à l'affichage, l'italique avec une emphase sémantique.

La balise font n'est plus utilisée, on va utiliser span associé à une classe.
